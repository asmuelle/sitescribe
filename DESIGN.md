# DESIGN.md — SiteScribe Technical & Product Design

## Thesis

Field pros lose 30–60 unbillable minutes per job writing reports, and the places where the
work happens — basements, rural routes, industrial buildings — are dead zones where every
cloud copilot shows a spinner. SiteScribe turns photos and dictation into a structured,
reviewable draft report on-device, at zero signal and zero inference cost, before the
inspector leaves the site. Incumbents (Spectora, CompanyCam) do forms and photos but assume
connectivity; the defensible slice is structured AI extraction that works offline, wrapped
in a trust layer that makes it safe for liability-bearing documents.

## Architecture

### Pipeline (capture → on-device inference → store → surface)

```
Camera / Mic
   │  CaptureKit: persist to GRDB FIRST (invariant 6), enqueue processing
   ▼
RecognitionKit  — deterministic layer, all devices
   │  Vision RecognizeDocumentsRequest (OCR), barcode, regex parsers
   │  (serials, model numbers, meter values, dates, units, prices)
   ▼
ExtractionKit   — on-device LLM layer
   │  FoundationModels @Generable against template-derived structs
   │  AFM 3 Core (text) everywhere; AFM 3 Core Advanced + OCRTool image
   │  input ONLY behind DeviceCapabilities check (invariant 3)
   │  Transcripts chunked to fit ~4k context, merged (invariant 7)
   ▼
StorageKit      — offline-first store + local RAG
   │  GRDB/SQLite; sqlite-vec + NLContextualEmbedding index over past reports
   ▼
ReviewFeature   — trust gate: provenance + confidence per field, confirm/edit
   ▼
ReportKit       — template assembly, PDFKit render, CSV export
   ▼
SyncKit         — BGTaskScheduler queue drained on reconnect;
                  opt-in cloud polish pass (Claude Haiku-class / Gemini Flash)
```

### Cost discipline: deterministic → on-device → frontier

| Layer | Used for | Cost |
|---|---|---|
| Deterministic (Vision OCR, barcode, regex) | Every identifier, number, date, unit | Free, exact, universal |
| On-device LLM (AFM 3 via @Generable) | Classification (defect/severity), narrative drafting, transcript structuring | Free, zero COGS at unlimited volume — this funds the generous free tier |
| Frontier cloud (Haiku-class) | Opt-in polish of narrative prose on sync, never extraction of facts | ~$0.01/report, off the critical path |

The adversarial review is right that cloud-on-sync could do extraction more accurately;
our answer is that same-visit value (re-shoot validation of a misread plate while still
standing in front of it, instant draft before leaving site, offline "what did I flag here
last time?" RAG) plus zero-COGS economics is the edge slice — so the architecture keeps
facts deterministic, drafting local, and polish optional.

### Modules

The SPM package map is normative in AGENTS.md; app shell is generated by XcodeGen from
`project.yml`. SyncKit is the only module permitted to import a networking client —
enforce with a lint/CI grep and a network-denying URLProtocol in all other modules' tests.

## Data Model Sketch

- **Job** — id, clientName, siteAddress, jobType (inspection/adjustment/service/receiptRun),
  status, visitDates[], templateId
- **CaptureItem** — id, jobId, kind (photo/audio/documentScan), fileURL, capturedAt,
  location?, deviceCapabilitySnapshot, processingState (persisted/recognized/extracted/failed)
- **Transcript** — id, captureItemId, fullText, segments[{text, startMs, endMs, confidence}],
  engine (speechAnalyzer/whisperCpp)
- **RecognizedArtifact** — id, captureItemId, kind (ocrBlock/barcode/serial/meterValue/price/date),
  rawValue, normalizedValue, boundingRegion, parserId, confidence
- **ReportTemplate** — id, name, version, jsonSchema (source of truth, compiled to
  @Generable structs), sections[], isFirmShared
- **Report** — id, jobId, templateId+templateVersion, status (draft/inReview/finalized),
  finalizedAt?, pdfURL?, polishState (none/requested/applied)
- **ExtractedField** — id, reportId, schemaKeyPath, value, source (deterministic/llm/manual),
  provenance {captureItemId, region?/timespan?}, confidence, reviewState
  (unreviewed/confirmed/edited) — the unit of trust; invariant 5 lives here
- **RAGChunk** — id, sourceRef (reportId/captureItemId), text, embedding (sqlite-vec),
  jobId, createdAt
- **SyncOperation** — id, entityRef, operationKind, payloadHash, state
  (queued/inFlight/done/failed), attempts, lastError?, requiresUserOptIn (cloud polish)

## Key Flows

### 1. Offline capture → draft report (basement walkthrough)

1. Inspector opens Job, taps the safety-orange capture button; airplane-mode badge shows "offline — everything still works".
2. Photos and a running voice note are written to StorageKit immediately (invariant 6); UI never blocks on processing.
3. RecognitionKit OCRs each photo and parses serials/meter values deterministically; Transcript is produced on-device.
4. ExtractionKit chunks the transcript and fills the template's @Generable structs: defect, location, severity, recommendedAction per finding.
5. A draft Report appears with every field carrying a provenance chip and confidence tint — before the inspector leaves the site.

### 2. Serial plate read with progressive enhancement

1. User scans a corroded furnace plate with the VisionKit scanner (universal path).
2. RecognitionKit returns the serial with low confidence; on AFM-Advanced-capable devices, ExtractionKit additionally runs image-input cross-checks via OCRTool.
3. Confidence still low → the field is flagged amber and the UI prompts an on-site re-shoot with framing guidance (this is the moment cloud-on-sync cannot serve).
4. The accepted value is stored as a RecognizedArtifact and copied verbatim into the report (invariant 4) — the LLM never touches the serial.

### 3. Review & finalize (trust gate)

1. ReviewFeature lists every ExtractedField grouped by section; unreviewed AI fields are visually distinct (amber) from deterministic (mono-spaced, sourced) and confirmed (green) ones.
2. Tapping a field shows its provenance: the source photo region or transcript span, plus confidence.
3. Finalize is disabled until no field has `reviewState == unreviewed` (invariant 5).
4. ReportKit renders the PDF with the firm's branding; the inspector signs; CSV export available alongside. The app never auto-sends.

### 4. Reconnect sync + optional polish

1. Connectivity returns; BGTaskScheduler drains SyncOperations in order, with retry/backoff.
2. If the user opted in for this report, SyncKit sends narrative fields only (never raw captures by default) for a Haiku-class polish pass and stores the diff for one-tap accept/reject.
3. Edits made on-device while queued merge via last-writer-wins per field with a CRDT-ish vector for sections; conflicts surface in ReviewFeature, never silently overwrite.

### 5. "What did I flag here last visit?"

1. On opening a Job at a repeat site, the inspector asks the question in the search bar (or taps the suggestion chip).
2. StorageKit runs a sqlite-vec similarity query over RAGChunks scoped to the site, fully offline.
3. Results render as cards linking to the original ExtractedFields and photos — answers are citations to past reports, never generated claims.

## Product & Visual Design Direction: "Jobsite Instrument"

SiteScribe should look like a rugged field instrument, not a SaaS dashboard: the design
reference is a quality multimeter or thermal camera UI. Gloved hands, direct sunlight,
dark basements, one-handed use.

- **Palette:** asphalt charcoal surfaces (`oklch(22% 0.01 260)`) with paper-white report
  surfaces; **safety orange** (`oklch(68% 0.19 45)`) reserved for capture/record actions;
  amber strictly means "unreviewed AI field", green strictly means "confirmed/synced".
  Color is semantic state, never decoration (no gradients, no blobs).
- **Typography:** SF Pro for UI; **SF Mono for every captured value** (serials, meter
  readings, prices) to signal "verbatim from the device, not generated" — the trust layer
  made visible. Large, condensed-feel section headers; minimum 17pt body, oversized
  glove-friendly tap targets (≥56pt) on capture surfaces.
- **Texture/depth:** flat, high-contrast field screens for sunlight; report/review screens
  switch to a calmer paper-white editorial layout — the moment of professional judgment
  reads differently from the moment of capture. iOS 26 Liquid Glass only on system chrome,
  never on data.
- **Motion:** progress is shown as pipeline state (persisted → recognized → extracted),
  not spinners; offline state is a quiet badge, never a blocking modal.

## Milestones

### M0 — Bootstrap (make `just ci` green)

- `project.yml` (XcodeGen): app target `SiteScribe`, iOS 26 deployment target, local SPM
  packages per the module map, `SiteScribe` scheme with test plan.
- Empty-but-compiling packages, one placeholder Swift Testing test per package;
  `.swiftlint.yml` + `.swiftformat` configs committed.
- **Accept:** `just bootstrap && just ci` passes locally and on the macos-15 CI runner
  (guard now takes the bootstrapped branch).

### M1 — Thin vertical slice (one template, one device, fully offline)

- Hardcoded "Home Inspection — General" template (JSON Schema → @Generable Finding struct:
  defect, location, severity, recommendedAction).
- Flow: create Job → photo capture persists to GRDB → Vision OCR → ExtractionKit fills
  Finding(s) → ReviewFeature confirm → PDFKit export. Single device class, no sync, no audio yet.
- ExtractionEngine protocol with deterministic fake; golden tests for 10 fixture photos.
- **Accept:** the full flow completes in airplane mode on an iPhone 16 simulator; offline
  test suite (network-denying URLProtocol) green; extraction golden tests green on CI via the fake.

### M2 — Trust layer

- Provenance chips + confidence tints on every field; finalize gate on `unreviewed` fields;
  dictation with SpeechAnalyzer + chunked extraction; durability tests (kill mid-capture);
  no-egress proof: CI grep forbids networking imports outside SyncKit + runtime
  `SITESCRIBE_DISABLE_NETWORK` assertion suite; AFM-capability progressive enhancement with
  graceful text-only behavior on older devices.
- **Accept:** invariants 1–7 each have at least one automated test; a finalized PDF shows
  per-field source attribution in an audit appendix.

### M3 — Monetization wiring

- StoreKit 2: Solo $16.99/mo / $149/yr; free tier metered at 10 AI-extracted documents/mo
  (deterministic capture stays unlimited — the wedge); Teams tier flagged behind a feature
  flag pending shared-template backend.
- Paywall at the extraction step (never at capture or export of already-finalized reports —
  invariant 8 means we never ransom existing data).
- **Accept:** sandbox purchase/restore tests green; metering counts only AI extractions;
  free-tier user can still capture, review deterministic fields, and export.

## Risks & Mitigations (from the adversarial review)

| # | Risk | Mitigation |
|---|---|---|
| 1 | "Offline AI" conflated with offline capture — cloud-on-sync at ~$0.01/job beats on-device accuracy | Lead with same-visit value (re-shoot validation, instant on-site draft, offline RAG) and zero-COGS free tier; ship hybrid polish pass so we win on both axes |
| 2 | Hardware floor: AFM 3 image input needs iPhone 15 Pro+; field fleets run older/rugged devices | Universal Vision OCR path on all devices is the baseline (invariant 3); AFM image input is enhancement only; test matrix includes a text-only-AFM device profile |
| 3 | Hallucinated severity/serial in an E&O-liability report — worst possible failure | Deterministic readers own all identifiers/numbers (invariant 4); provenance + confidence + mandatory review gate (invariant 5); app never auto-sends |
| 4 | Incumbents copy the pipeline — primitives are now free system APIs; Spectora ships AI Comment Assist and owns the suite | Single vertical (home inspectors first), export INTO incumbent formats (Spectora-importable) instead of replacing the suite; firm template libraries as the lock-in asset; speed |
| 5 | Distribution: not a search category; suites are sold via conferences/franchise deals | Bottom-up solo wedge (CompanyCam playbook): generous free tier funded by zero COGS, receipt/invoice capture for solo contractors, InterNACHI community content before any sales motion |

Secondary watch items: thermal throttling on 8-hour shifts (budget sustained
camera+ASR+LLM sessions, degrade to deferred extraction), ASR quality under HVAC/generator
noise (whisper large-v3-turbo fallback + jargon custom vocabulary), and Android timing
(ML Kit path is universal; do not wait on AICore preview maturity).
